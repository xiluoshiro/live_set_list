# Frontend 全局样式漏检复盘

标签：`TODO` `待实现`

本文档记录 2026-04-12 这轮控制台试点 antd 时，为什么“引入全局样式”没有被现有测试及时拦截。

## 1. 问题现象

- 在入口文件引入了全局样式（`antd/dist/reset.css`）后，非控制台页面样式也发生了变化。
- 现有测试（主要是 `ConsoleInsertPanel` 行为测试）仍然全部通过，未报警。

## 2. 没覆盖到的根因

1. 现有前端测试以“功能行为断言”为主，不校验跨页面视觉与样式回归。
2. 当前控制台测试范围集中在组件内部交互，不覆盖 `main.tsx` 的全局样式导入副作用。
3. `jsdom` 环境对 CSS 视觉结果不敏感，无法发现“页面外观被全局 reset 影响”的问题。

## 3. 没看护到的场景

1. 在入口（如 `main.tsx`）新增/修改全局 CSS 导入。
2. 引入第三方 UI 库的 reset/base 样式，导致全站默认元素表现变化。
3. 非当前改动页面（例如主列表、详情弹窗）出现视觉退化，但组件测试仍通过。
4. 局部试点改动误用全局样式策略，造成“试点影响全局”。
